Class: FeedGenerator

Defined in: models/feed-generator.coffee
Inherits: EventEmitter

Overview

This model takes care of all the tasks required to generate the raw XML data that are then written to a file. Note that it does not directly write the file, instead it emits a 'feedgenerated' event with the XML string.

Examples:

  generator = new FeedGenerator 'hindawi', {
      host: 'http://www.hindawi.org/',
      url: 'http://www.hindawi.org/books',
      language: 'ar',
      selectors: {
          ...
      }
  }, './feeds/hindawi.xml'

  generator.on 'feedgenerated', (xml) ->
      fs.writeFile xml, done

  generator.on 'initialized', ->
      generator.generate()

  generator.initialize()

Events:

initialized

  • Emitted when the generator has checked for a cached feed and finished processing it.

feedgenerated

  • Emitted when the whole XML data is ready

error

Property Summary

(Number) maxPages
The maximum number of pages to load, each page instantiates a new PageLoader
(Boolean) forceLimit
Whether to honor the maxPages limit even in cases where there are more pages to load until lastArticleUrl is matched. Setting this to true may cause some articles to be missing from the feed.

Instance Method Summary

  • # (void) initialize() This method checks for a cached feed and waits until the cache is processed and all cached items are added to the new @feed.
  • # (void) generate() Generates the feed as XML, emits a feedgenerated event on finish with the XML data

Constructor Details

Parameters:

Instance Method Details

# (void) initialize()

This method checks for a cached feed and waits until the cache is processed and all cached items are added to the new @feed. You should call this after adding listeners to the initialized event.

# (void) generate()

Generates the feed as XML, emits a feedgenerated event on finish with the XML data
